# Langfuse — LLM Observability 공부 자료

> 수집일: 2026-05-30 | 태그: `#observability` `#langfuse` `#tracing` `#evaluation`

## 한줄 요약

Langfuse = LLM 앱의 "로그 + 평가 + 프롬프트 관리"를 한 곳에서. 오픈소스, self-host 가능.

---

## 핵심 개념

### Trace 구조
```
Trace (요청 단위)
├── Span (중간 처리 단계)
│   ├── Generation (실제 LLM 호출)
│   └── Span (툴 호출, 검색 등)
└── Score (평가 점수 attach)
```

- **Trace**: 하나의 사용자 요청 전체
- **Span**: 처리 단계 (RAG 검색, 툴 호출 등)
- **Generation**: LLM 호출 1회 (토큰, 비용, latency 자동 기록)
- **Score**: 평가 점수 (LLM-as-judge, 사용자 피드백, 자동 평가)

---

## 주요 기능

### 1. Tracing (관찰)
- Decorator API: `@observe()` 한 줄로 함수 자동 추적
- Manual API: 세밀한 제어 필요할 때
- OpenTelemetry 호환: 기존 OTel 파이프라인 연동 가능
- 지원 통합: LangChain, OpenAI SDK, LiteLLM, Anthropic SDK, vLLM

### 2. Evaluation (평가)
- **LLM-as-a-judge**: 평가 rubric 정의 → LLM이 자동 채점
- **Dataset experiments**: 테스트셋 만들고 여러 프롬프트/모델 비교
- **Human annotation**: 팀원이 직접 라벨링
- **Custom scores**: 코드로 직접 점수 계산해서 attach

### 3. Prompt Management
- 프롬프트 버전 관리 (Git처럼)
- 프로덕션 프롬프트를 코드에서 분리
- A/B 테스트 지원

### 4. Cost & Latency 모니터링
- 모델별/사용자별/feature별 토큰 사용량 집계
- p50/p95/p99 latency 대시보드

## 2026년 주목할 업데이트

- **Self-host ClickHouse 네이티브**: 대용량 trace 저장에 최적화, 단일 컨테이너 → 분산 가능
- **LLM-as-judge 실행 트레이싱**: judge LLM 호출 자체도 trace로 기록 → 평가 품질 디버깅 가능
- **vLLM 통합 강화**: self-hosted vLLM + Langfuse 풀 파이프라인 가이드

## Cloud vs Self-host 선택 기준

| | Cloud | Self-host |
|--|-------|-----------|
| 설정 | 즉시 사용 | Docker Compose 5분 |
| 데이터 주권 | Langfuse 서버 | 완전 내부 |
| 비용 | 사용량 기반 | 인프라 비용만 |
| 적합 | 빠른 프로토타입 | 프로덕션, 규제 환경 |
